# Local Agent

The Local Agent runs near the user's data, usually on a local workstation, lab server, or HPC login environment. It executes approved commands, streams logs, computes hashes, and returns artifact metadata.

## Core Principle

A local agent must not execute arbitrary shell commands generated by an LLM.

It should execute only:

- reviewed command templates declared in manifests or workflow templates;
- reviewed wrapper submissions declared in protocol artifacts;
- explicit internal validation steps that do not invoke scientific tools.

## Approval Boundary

Before running a command, the agent should show:

- selected workflow;
- selected manifest;
- readiness level;
- rendered command or wrapper submission;
- input paths;
- expected output paths;
- model-weight source and checksum status;
- license and citation metadata;
- hardware and scheduler request;
- run directory;
- limitations and non-claims.

Execution should not proceed when approval is required but absent.

## Refusal Is Normal

A correct local agent may refuse execution when:

- required inputs are missing;
- license or citation metadata is unclear;
- model-weight checksum is missing;
- readiness is below the requested execution mode;
- the wrapper script is not reviewed;
- the command contains unreviewed shell composition;
- the task asks for clinical, drug-efficacy, or experimental-validation claims.

## Run Record

Every attempted execution should produce a run record when possible.

At minimum, record:

- rendered command;
- command source;
- return code;
- input hashes;
- output artifacts;
- tool versions;
- environment;
- logs;
- citations;
- licenses;
- limitations;
- warnings.
