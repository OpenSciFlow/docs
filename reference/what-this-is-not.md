# What This Is Not

OpenSciFlow needs clear boundaries.

## Not a Mature Standard

The current repositories are public drafts.

They are meant for correction, review, and small experiments, not production claims.

## Not a General AI Scientist

OpenSciFlow does not try to automate scientific discovery end to end.

It focuses on making existing tools and models easier to inspect, check, run under bounded conditions, cite, record, and diagnose.

## Not a Run-Anywhere System

OpenSciFlow does not guarantee that tools run across all environments.

Capsules can only claim evidence for environments recorded in their verified environment matrix.

## Not a Workflow Engine Replacement

OpenSciFlow should interoperate with existing systems such as:

- Nextflow;
- Snakemake;
- Galaxy;
- CWL;
- WDL / Cromwell;
- AiiDA;
- Parsl;
- Slurm.

It does not replace them.

## Not a README Replacement

OpenSciFlow capsules do not replace upstream documentation.

They capture execution-facing requirements and evidence that an agent must check before acting.

## Not a Clinical or Drug-Development Decision System

OpenSciFlow outputs should not be used as clinical, diagnostic, drug-efficacy, toxicity, or regulatory conclusions.

Computational outputs are not experimental validation.

## Not a Partnership Claim

Listed projects are related projects only.

OpenSciFlow does not claim partnership, endorsement, or official association unless maintainers explicitly agree.

## Not Permission for Arbitrary Execution

A local agent should not execute arbitrary shell generated by an LLM.

It should execute only reviewed command templates or reviewed wrapper submissions declared in protocol artifacts.
